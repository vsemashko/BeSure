## Post-Submission

### Review Timeline

**Typical Timeline:**
- Submission: Immediate
- "Waiting for Review": 1-3 days
- "In Review": 1-24 hours
- Decision: Approval or Rejection

**Total Time:** Usually 1-5 days

### During Review

**Monitoring:**
- Check App Store Connect daily
- Monitor email for updates
- Keep contact phone available
- Be ready to respond to questions within 24 hours

**Status Meanings:**
- **Waiting for Review:** In queue
- **In Review:** Being actively reviewed
- **Pending Developer Release:** Approved, waiting for you to release
- **Ready for Sale:** Live on App Store
- **Rejected:** See resolution steps below

### If Approved

**Next Steps:**
1. Receive approval email
2. Choose to release immediately or schedule
3. App goes live (or at scheduled time)
4. Monitor for crashes and reviews
5. Respond to user reviews
6. Track analytics

**Post-Launch:**
- Monitor crash reports (Sentry)
- Watch App Store reviews
- Track download numbers
- Collect user feedback
- Plan version 1.1 improvements

### If Rejected

**Common Rejection Reasons:**
1. **Guideline 2.1** - App completeness (bugs, crashes)
2. **Guideline 4.3** - Spam, copycats
3. **Guideline 5.1.1** - Privacy issues
4. **Guideline 2.3.1** - Hidden features
5. **Guideline 5.2.1** - Legal requirements

**Resolution Steps:**
1. Read rejection reason carefully
2. Understand the specific guideline violated
3. Fix the issue in code (if needed)
4. Update metadata (if needed)
5. Respond in Resolution Center
6. Resubmit

**Typical Response Time:**
- Fix and resubmit: Same day
- Back in review: 1-2 days
