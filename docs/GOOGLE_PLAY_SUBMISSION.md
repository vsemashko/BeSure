- Instant distribution
- Use for team testing

- Up to 100,000 testers
- Requires email list or Google Group

- Unlimited testers

- All users

1. **Internal Testing:** Test with team (1-2 days)

## App Bundle Upload

### Creating Android App Bundle (AAB)

**Using Expo/React Native:**

```bash
cd mobile
npx eas build --platform android --profile production
```

**Using Android Studio:**
1. Build ‚Üí Generate Signed Bundle / APK
2. Select "Android App Bundle"
3. Create or select keystore
4. Build release bundle

**Requirements:**
- Signed with upload key (or app signing key)
- Version code incremented
- Version name updated
- ProGuard enabled (recommended)
- Target latest API level

### Upload Process

1. Go to Release ‚Üí Production ‚Üí Create new release
2. Click "Upload" in "App bundles" section
3. Select your .aab file
4. Wait for processing
5. Review warnings/errors (if any)
6. Add release notes
7. Proceed to review

### App Signing

**Play App Signing (Recommended):**
- Google manages your app signing key
- You upload with upload key
- More secure
- Enables key upgrade

**Self-Managed:**
- You manage app signing key
- Higher risk if key is lost

**Recommendation:** Use Play App Signing ‚úÖ

## Post-Submission

### Review Timeline

**Typical Timeline:**
- Upload: Immediate
- Processing: 10-30 minutes
- Review: 1-7 days (usually 1-3 days)
- Published: Immediate upon approval

**Status Meanings:**
- **In review:** Being reviewed by Google
- **Approved:** Passed review
- **Rejected:** Needs fixes
- **Published:** Live on Play Store

### After Approval

**Next Steps:**
1. Receive approval notification
2. App goes live (if 100% rollout)
3. Gradual rollout (if staged rollout)
4. Monitor for crashes and ANRs
5. Respond to user reviews
6. Track installs and ratings

**Post-Launch Monitoring:**
- Check Android Vitals (crashes, ANRs)
- Monitor Play Console ratings
- Read and respond to reviews
- Track install numbers
- Watch for policy warnings
- Collect user feedback

### If Rejected

**Common Rejection Reasons:**
1. Privacy Policy issues
2. Misleading content or metadata
3. Copyrighted content
4. Inappropriate content
5. Malware or security issues
6. Broken functionality
7. Spam or low-quality app

**Resolution Steps:**
1. Read rejection reason carefully
2. Fix the issue
3. Update app bundle (if code fix needed)
4. Update store listing (if metadata issue)
5. Reply to rejection explaining fixes
6. Resubmit for review
